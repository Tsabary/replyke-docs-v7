## Overview

`useIsEntityInCollection` provides a function to check whether a specific entity is saved in any collection owned by the current user. The result is stored in local state and also returned from the async call.

<Note>Requires an authenticated user. Throws if called without a logged-in user, an entity ID, or a project ID.</Note>

## Usage Example

```tsx
import { useIsEntityInCollection } from "@replyke/react-js";

function SaveStatus({ entityId }) {
  const { checkIfEntityIsSaved, entityIsSaved } = useIsEntityInCollection();

  useEffect(() => {
    checkIfEntityIsSaved({ entityId });
  }, [entityId]);

  if (entityIsSaved === null) return <p>Checking...</p>;
  return <p>{entityIsSaved ? "Saved" : "Not saved"}</p>;
}
```

## Returns

<ResponseField name="checkIfEntityIsSaved" type="({ entityId }: { entityId: string }) => Promise<boolean>">
  Calls the API to determine whether the entity is saved in any of the user's collections. Updates `entityIsSaved` state and returns the boolean result.
</ResponseField>

<ResponseField name="entityIsSaved" type="boolean | null">
  The last known saved status. `null` before the first check is completed.
</ResponseField>

<Note>
  For richer data (which specific collections contain the entity), use `isEntityInCollection` from `useCollections` instead — it returns the full list of matching collections.
</Note>
